Chainspotting: Queries

This module of Chainspotting allows you to process real-time blockchain data within a spreadsheet.

In addition to standard spreadsheet features, you can use the following functions, which will automatically update based on activity on the blockchain:

BTCADDR(addr,data_type,conf) Returns data about a bitcoin address referenced by addr. data_type can currently be "balance," "sent," or "received," and conf (either true or false) determines whether or not the values returned are confirmed amounts.
BTCTRN(hash,data_type) Returns data about a bitcoin transaction referenced by hash. data_type can currently be "confirmations," "amount," or "fees."
BTCOPRET(hash,data_type) Returns data about an OP-RETURN transaction referenced by hash. data_type can currently be "hex" or "text."
BTCFIND(addr,data_type,num) Returns a transaction associated with a bitcoin address referenced by addr. data_type can currently be normal "transactions" or "op-returns" and num determines which transaction you want. For example, if you want the latest transaction (or OP-RETURN transaction), enter 1.
BTCPRICE(curr) Returns the most current bitcoin price from blockchain.info for the specified curr. curr can currently be one of the following: AUD BRL CAD CHF CLP CNY DKK EUR GBP HKD ISK JPY KRW NZD PLN RUB SEK SGD THB TWD USD.

These functions operate on the main blockchain unless you select TESTNET3 from the Blockchain menu.

NOTE: To create a cell range for these functions (and the standards ones) use Shift while dragging.

No user data is saved on any server, nor are cookies used in any way. When you load or save a document, the app uses local storage. Keep in mind local storage may be limited on your system. It can also difficult to backup or restore data to and from it. Therefore, you should Export important files to your local file system. You can also later Import these files.

Custom development of this app and integration of it into your pipeline is available upon request, by clicking the contact button on the top of the screen. You can also view the module's source code on GitHub by clicking the source button.
